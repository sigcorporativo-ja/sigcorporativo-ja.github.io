Notas versión 6.3.0

API js

  • Actualización OpenLayer a v6.15.1
  • Los objetos Layer pasan a ser gestionados internamente mediante un nuevo atributo 'id' en lugar de mediante su atributo 'name', lo que permite tener en un mismo mapa diferentes capas con el mismo nombre. Esto permitiría por ejemplo cargar dos veces la misma capa desde un servicio WMS, cada una con un 'styles' diferente.
  • Las capas MVT pueden ser ahora capas "base" de un mapa.
  • Plugin printerdos ofrece un soporte limitado a capas MVT.
  • El nuevo parámetro de mapa 'layerswitcher' permite inicializar dicho control con una capa vacía como capa base.
  • Plugin searchstreetgeosearch: Los resultados del autocompletado ya no pisan los resultados de la búsqueda. Se borran resultados dibujados en el mapa al pulsar sobre "limpiar búsqueda".
  • Todos los plugins del core disparan ahora el evento M.evt.ADDED_TO_MAP al ser añadidos al mapa.
  • Las simbologías temáticas pasan a usar internamente la simbología Generic.
  • Nuevo método de mapa para cambiar a la escala más cercana.
  • Nuevo metodo de mapa para añadir más de un plugin a la vez.
  • Plugin attributetable ofrece ahora la posibilidad de seleccionar y hacer zoom a elementos desde la tabla de atributos.
  • Cuando no se especifique atributo 'version' de una capa WMS, se pasará a usar por defecto la versión 1.3.0 de dicho estándar.
  • Las propiedades de configuración de Mapea no dependen ya del protocolo del cliente, se establecen por defecto a https.
  • Si el sistema donde se levanta Mapea tiene configuradas propiedades java de autenticación de proxy para acceder a internet, el proxy de Mapea utilizará esas

El código fuente se encuentra disponible en GitHub, y la wiki se irá actualizando para recoger estas novedades en los próximos días.
