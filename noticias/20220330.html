Situación actual

Actualmente existen en Producción dos versiones del servicio de impresión en alta resolución Geoprint:

  • El servicio original, basado en una versión inicial de MapfishPrint, accesible desde: https://geoprint-sigc.juntadeandalucia.es/geoprint
  • El servicio actualizado a MapfishPrint v3, accesible desde: https://geoprint-sigc.juntadeandalucia.es/geoprint3

El concepto básico de ambas versiones es el mismo: atender peticiones de impresión de mapas a un DPI específico partiendo de la información necesaria para generarlos: servicios OGC de las capas raster, los datos de las vectoriales, coordenadas de la vista y dimensiones del formato.

Las mejoras de la v3 respecto a la anterior incluyen, además de las mejoras técnicas y de rendimiento propias de este tipo de evolutivo, aspectos tales como la posibilidad de diseñar las plantillas de impresión en formato jasper reports, y la posibilidad de disponer de las denominadas "configuraciones de impresión", que son ahora independientes unas de otras facilitando así su consumo por parte de los clientes.

Uso de Geoprint desde Mapea

Geoprint es un Servicio Web y puede consumirse como tal. Adicionalmente, se desarrolló un plugin de Mapea que ofrece la posibilidad de consumirlo desde un visualizador de mapas, llamado "printer".

    https://mapea4-sigc.juntadeandalucia.es/?printer

Desafortunadamente, la v3 de MapfishPrint no es retrocompatible, y es por ello que actualmente conviven las dos instancias del servicio y que se ha desarrollado un plugin nuevo compatible con la v3:

   https://mapea4-sigc.juntadeandalucia.es/?printerdos

De cara a facilitar todo lo posible la migración desde el plugin antiguo en aquellos visores que lo usen, este nuevo plugin es compatible con todas las versiones de Mapea.

Migración plugin de impresión en Mapea api js

Para migrar de un plugin a otro, basta con reemplazar los ficheros del plugin printer por los de printerdos, ya sea explícitamente:

  • https://mapea4-sigc.juntadeandalucia.es/plugins/printerdos/printerdos-4.1.0.ol.min.js
  • https://mapea4-sigc.juntadeandalucia.es/plugins/printerdos/printerdos-4.1.0.min.css

O mediante el api rest de resources-plugin (por ejemplo, para una versión 5.1.0 de Mapea, las llamadas serían):

  • https://mapea4-sigc.juntadeandalucia.es/mapea/api/actions/resourcesPlugins?name=printerdos&version=5.1.0&type=js
  • https://mapea4-sigc.juntadeandalucia.es/mapea/api/actions/resourcesPlugins?name=printerdos&version=5.1.0&type=css

Ejemplo funcional de Mapea v4.x con plugin printerdos:  

     https://jsfiddle.net/sigcJunta/p3b8hsk4/

Si existen plantillas específicas para el visor

Algunos visores de mapas que usan Geoprint disponen de su propia plantilla de impresión, con logotipo y formato personalizados. En esos casos, se ha migrado la plantilla del antiguo formato yaml al nuevo diseño jasper, y para acceder a ella desde el visor, basta con iniciar el plugin printerdos con la url asociada a dicha configuración. Más información en la página del plugin printerdos:

    https://github.com/sigcorporativo-ja/Mapea4/tree/develop/mapea-js/src/plugins/printerdos

Tal como se ha comentado antes, ahora el plugin solo mostrará las plantillas asociadas a esta configuración, por lo que si con el plugin anterior se realizaba alguna gestión de los desplegables para que eliminara otras plantillas, ya no es necesario.

Migración plugin de impresión en Mapea api REST

Para la migración del plugin en un Mapea generado por el api REST, solo hay que cambiar de "printer" a "printerdos" en la llamada al api:

    http://mapea4-sigc.juntadeandalucia.es/?printerdos
